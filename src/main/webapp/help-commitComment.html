This value is used as the template of Subversion commit message, and populated as the default when this plugin is enabled. You can embed the following variables with groovy string format.
env
Map returned by hudson.model.AbstractBuild.getEnvironment().
sys
Properties returned by System.getProperties().
For example, you can embed BUILD_ID in the Subversion commit comment. 
        Commit by Jenkins svn-commit-plugin. Build: ${env['BUILD_ID']}
